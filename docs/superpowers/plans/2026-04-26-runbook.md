## Step 3 — Ranking sanity check (GPU, ~30 min)

Re-ranks the freeexcl model with our `rank_features.py` and compares its top-20 tool-signed features against the zip's gold CSV. Per spec §13 we tolerate ≤5 swaps (≥15/20 overlap). Catches any divergence in prompt sampling, encoding, or scoring before we waste GPU time generating the other two CSVs.

```bash
source /home/cs29824/.venv/bin/activate
cd /home/cs29824/andre/xcdr3

python rank_features.py --crosscoder antebe1/dfc-D8k-excl10-freeexcl-k160 --out data/rankings/_sanity.csv

python -c "
import pandas as pd
ours = pd.read_csv('data/rankings/_sanity.csv')
gold = pd.read_csv('data/rankings/dfc-D8k-excl10-freeexcl-k160.csv')
ours_top = set(ours[ours['sign']=='tool'].nlargest(20,'cohens_d')['feature_idx'])
gold_top = set(gold[gold['sign']=='tool'].nlargest(20,'cohens_d')['feature_idx'])
print(f'overlap: {len(ours_top & gold_top)}/20')
assert len(ours_top & gold_top) >= 15, 'overlap below tolerance — investigate'
"

rm data/rankings/_sanity.csv
```

Expected: `overlap: N/20` with N ≥ 15.

## Step 5 — Smoke-test the sweep on 2 prompts (GPU, ~5 min)

Runs the full P1 driver end-to-end on 2 prompts × 48 cells before committing to the real 100-prompt sweep. Verifies that loading + encoding + patching + scoring all wire up correctly and that the α=1 invariant holds (`(α-1)·…` is zero, so the steered patch reduces to `h_a`, and the model A generation should match the clean run).

```bash
source /home/cs29824/.venv/bin/activate
cd /home/cs29824/andre/xcdr3

mkdir -p results/targeted_steering/_smoke
python run_steering_eval.py \
    --crosscoder antebe1/dfc-D8k-excl10-freeexcl-k160 \
    --rankings  data/rankings/dfc-D8k-excl10-freeexcl-k160.csv \
    --out       results/targeted_steering/_smoke \
    --n-prompts 2 \
    2>&1 | tee logs/smoke.log

python -c "
import json
with open('results/targeted_steering/_smoke/k01_a01.jsonl') as f:
    for line in f:
        r = json.loads(line)
        print('clean=', r['clean_score']['overall_score'],
              'recon=', r['recon_score']['overall_score'],
              'steered=', r['steered_score']['overall_score'])
"

rm -rf results/targeted_steering/_smoke
```

Expected: `k01_a01.jsonl` has 2 lines; `steered == clean` (or off by at most one rubric category from fp16/fp32 noise). If `steered` diverges from `clean`, audit `generate_with_patch` dtype handling before launching the full sweep.

## Recovery cheatsheet

| Symptom | Recovery |
|---|---|
| Sweep crashed mid-cell | Re-run the same `run_steering_sweep.sh` command — prompt-level resume picks up at the last completed prompt. |
| Smoke step's α=1 row shows `steered ≠ clean` | Inspect `generate_with_patch` (sweep_eval.py:322) for dtype mismatches; the patch should re-cast back to float16 (it already does at line 334). |
| Step 3 overlap < 15/20 | Compare `_sample_prompts` vs the zip's prompt sampling code (zip path: `model_diffing_crosscoders-main/neuron_identification/`). The seed (42) and split (`test`) should match. |
| OOM in step 4/5/6 | Lower `MAX_NEW_TOKENS` in `sweep_eval.py:66` from 200 → 128. |
| `models.jsonl` missing the 3 D8k entries | Re-copy from `/tmp/xcdr_zip/model_diffing_crosscoders-main/models.jsonl`; verify with `grep -E '"name": "antebe1/(dfc-D8k-excl10-freeexcl-k160\|cc-D8k-nol1-k45\|dfc-D8k-excl10-k45)"' models.jsonl \| wc -l` → 3. |
